# Contributing

This project is designed for URL-first paper sharing. Contributors should not
need to understand the internal YAML format, weekly issue names, or canonical
taxonomy before recommending a paper.

## Recommend a Paper

Use the Submit page on the website or open a paper suggestion issue and paste
the paper URL. The website form only opens a prefilled GitHub issue; it does not
write to the repository.

Required:

- Paper URL.

Optional:

- One short note explaining why the paper is relevant.
- Free-text tags such as `enzyme design`, `PLM`, or `wet lab validation`.
- Code, project, dataset, or benchmark link.

The automation will generate a metadata preview for maintainers. It will try to
detect title, authors, DOI, source, publication date, and suggested canonical
tags when possible.

## Maintainer Review

Maintainers review the issue preview and use labels as the curation interface:

- `accepted`: include the paper in the accepted archive.
- `featured`: mark it as a Pick of the Week candidate.
- `needs-info`: ask the submitter for more context.
- `rejected`: decline the suggestion.

When `accepted` is applied, GitHub Actions creates a curation pull request with
the generated paper YAML and regenerated website pages. Weekly issues are
derived automatically from `accepted_at`, so maintainers do not need to write
`YYYY-Www`-style files.

## Safety Boundary

- Public visitors cannot write to `data/papers/` from the website.
- The website does not hold a GitHub token.
- Paper suggestions are GitHub issues tied to the submitter account.
- Automation rejects local, private-network, and non-public URLs.
- Accepted records are reviewed through pull requests before they reach `main`.

## Manual Fallback

If automation is unavailable, maintainers can still add one YAML file under
`data/papers/YYYY/`. Use `templates/paper.yml` as a reference, then run:

```bash
python3 scripts/validate_papers.py
python3 scripts/build_docs.py
python3 -m unittest discover -s tests
```

## Do Not

- Do not upload PDF files.
- Do not copy abstracts into `why_it_matters`.
- Do not add broad protein design papers unless enzyme relevance is explicit.
- Do not edit generated pages directly unless you are changing the generator.

## Generated Files

The following files are generated by `scripts/build_docs.py`:

- `README.md`
- `docs/index.md`
- `docs/archive.md`
- `docs/info.md`
- `docs/assets/site.css`
- `docs/assets/app.js`
